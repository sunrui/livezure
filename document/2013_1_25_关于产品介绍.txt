关于 rpc network interaction engine。

0. 简单易用，占用网络空间极低（因为基于压缩二进制 based）。
1. 全程通讯数据标准 tea/aes 128 bit 加密（可选在特殊的交互中，如登陆账户，或全程）却保通讯安全。
   （关于 tea/aes/des/rsa 等加密技术对比，可参照百科）
2. 全程单 socket 动作，强大的粘包、断包、碎包处理引擎，自带校验，确保数据通讯无误。
3. 两级自定义交互 id，访问远程就要调用本地 api 一样简单。
4. 核心多级缓存池诸级碰撞。
（
整个内存动作：
接收池->预加载池空间-->适应应用数据块预装载 -->合适打包加密池空间-->发送对列池--->
底层实在不够需要页面交换时-->从内存池中抽出来固定的块重复使用-->内存池 --> 实在没有时 malloc
）
（可以测试百万次服务器交互，内存跳动丝毫未动，稳定在 2m-3m 左右。）
5. 自动变长的 rpc 交互数据代替传统定长的处理模式，既省带宽又减轻服务器压力。
5. 纯 c 内核设计，全程异步处理，体积最小、占用内存极低，却毫不失性能。
   （服务器端、客户端内存 2-3m，体积 150kb）。
6. 稳定的心跳机制，及时处理客户端断开连接、以及长时间未操作情况（如果有需求见下）。
(应用场景：用户在打开一个页面时，会动态的与服务器作为交互响应，如果用户长时间未操作请求，将在用户唤醒的时候要求用户输入认证等其它处理，大大提高了通讯安全。）
7. 跨全平台 windows, linux, bsd, mac支持。内核 IOCP/EPOLL/KQUEUE，却保 IO 输出最大化。
6. 服务器独有的 push 功能，将消息实时投递给客户端。

测试用例，
千人网吧管理平台，采用了 livezure 网络交互引擎，彻底解决了，通讯单一，可控制性差，心跳机制复杂，加密、及需求扩展难的问题。
千百级并发数据处理，仅仅在数分钟既可完成，无一造成的错包，丢包等问题。
